# RPG Project: Implementing interface, sounds, and architectural patterns in Unity

## Описание проекта
В рамках данной практики в игру был добавлен пользовательский интерфейс. Проект демонстрирует применение паттерна **Entrypoint** (глобальный и для каждой сцены), разделение логики по уровням абстракции («сверху вниз»), а также реализацию UI через **MVC** (Model-View-Controller). Добавлены звуковые эффекты (фоновая музыка, звуки наведения и нажатия на кнопки), управление громкостью.

## Введенные изменения

### 1. Доработки интерфейса
- **Главное меню**:
  - Кнопка **«Играть»** — загружает игровую сцену `Gameplay`.
  - Кнопка **«Настройки»** — открывает панель настроек со слайдерами громкости.
  - При наведении на кнопки проигрывается звук, при нажатии — другой звук.
- **Игровое меню**:
  - Во время игры по нажатию `ESC` открывается меню с кнопками:
    - **«Продолжить»** — возврат в игру.
    - **«Главное меню»** — возврат в `MainMenu`.
    - **«Сохранить»** — сохраняет текущее состояние (позиция игрока, здоровье) в JSON-файл.
    - **«Загрузить»** — загружает последнее сохранение.
  - Аналогичная звуковая обратная связь для кнопок.

### 2. Архитектурные паттерны и принципы
- **Паттерн Entrypoint**:
  - Глобальный `GameEntrypoint` (в сцене `Initialization`) создаёт общие сервисы (`IAudioService`, `ISaveService`, `ISceneLoader`) и обеспечивает их доступность через синглтон. Он не уничтожается при смене сцен.
  - Для каждой сцены создан свой entrypoint (`MenuEntrypoint`, `GameplayEntrypoint`), который получает сервисы из глобального контекста, настраивает объекты сцены и создаёт соответствующие контроллеры.
- **Архитектура «сверху вниз»**:
  - **Верхний уровень** — конкретные реализации сервисов (`AudioService`, `SaveService`, `SceneLoader`).
  - **Средний уровень** — entrypoint сцен, которые используют интерфейсы сервисов и настраивают логику сцены.
  - **Нижний уровень** — компоненты геймплея (например, `Health`) и UI-элементы, управляемые через MVC.
- **MVC для интерфейса**:
  - **View** — MonoBehaviour, содержит ссылки на элементы UI и генерирует события (клики, изменение слайдера, наведение).
  - **Controller** — plain C# класс, подписывается на события View, взаимодействует с сервисами и обновляет Model/View.
  - **Model** — простые C# классы, хранящие состояние (например, `SettingsModel`, `GameModel`, которые будут реализованы в рамках следующей практики при интегрировании сохранения состояния игры (например, позиция и здоровье игрока)).


## Структура проекта

### Сцены
1. **Initialization** — загружается первой. Содержит объект `GameEntrypoint`, который инициализирует сервисы и переходит в `MainMenu`.
2. **MainMenu** — главное меню. Содержит `MenuEntrypoint` и `MenuView` на Canvas.
3. **Gameplay** — игровая сцена. Содержит `GameplayEntrypoint`, `GameplayView` (панель меню), игровые объекты (игрок, враги и т.п.).

### Основные компоненты

#### Глобальный Entrypoint (`GameEntrypoint`)
- Создаёт и хранит экземпляры сервисов.
- Помечен `DontDestroyOnLoad`.
- Загружает сцену `MainMenu` после инициализации.

#### Сервисы (интерфейсы и реализации)
- **`IAudioService` / `AudioService`** — управление фоновой музыкой и звуковыми эффектами. Использует два `AudioSource` (для музыки и SFX). Реализует регулировку громкости и сохранение настроек в `PlayerPrefs`.
- **`ISaveService` / `SaveService`** — сохранение/загрузка данных в JSON-файлы в `Application.persistentDataPath`.
- **`ISceneLoader` / `SceneLoader`** — обёртка над `SceneManager` для загрузки сцен (синхронной и асинхронной).

#### MVC для UI
- **`MenuView`** — содержит ссылки на кнопки и слайдер главного меню, объявляет события (`OnPlayClicked`, `OnSettingsOpened`, `OnMusicVolumeChanged`, `OnPlayButtonHover` и т.д.).
- **`MenuController`** — подписывается на события `MenuView`, обрабатывает нажатия, обновляет громкость через `IAudioService`, запускает звуки, загружает сцену по кнопке «Играть».
- **`GameplayView`** — аналогично, но для игрового меню (кнопки Resume, Save, Load, MainMenu) с обработкой нажатия `ESC`.
- **`GameplayController`** — связывает `GameplayView` с сервисами сохранения и загрузки, управляет паузой, вызывает загрузку сцен.
- **`GameModel`** — будет содержать ссылку на игровой объект игрока и используется для передачи данных между контроллером и сервисами.

#### Звуковое сопровождение
- В `AudioService` добавлены методы `PlayMusic`, `PlaySFX`, `StopMusic`.
- При инициализации сцены `MenuEntrypoint` и `GameplayEntrypoint` передают соответствующие аудиоклипы (фоновая музыка, звук наведения, звук клика) в конструкторы контроллеров.
- Для каждой кнопки настроен `EventTrigger` (PointerEnter), вызывающий методы `On...ButtonPointerEnter` во View, которые генерируют события для контроллера, а тот проигрывает звук через `IAudioService`.

## Итоги
В результате работы реализовано:
- Полноценное главное меню и меню паузы с сохранением/загрузкой.
- Гибкая архитектура с использованием паттернов Entrypoint и MVC.
- Чёткое разделение ответственности между уровнями приложения.
- Интеграция звукового сопровождения с возможностью регулировки громкости.
