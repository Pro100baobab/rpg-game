# RPG Project: Saving and loading (Repository-Interactor pattern)


## Описание изменений
В проект добавлена система сохранения и загрузки состояния игры. При нажатии на кнопку **«Сохранить»** в игровом меню сохраняются следующие данные:
- позиция игрока на карте;
- текущее здоровье (HP) игрока;
- оставшееся время перезарядки (кулдаун) магической атаки;
- позиции всех мобов (врагов) на карте;
- текущее здоровье каждого моба.

Кнопка **«Загрузить последнее сохранение»** восстанавливает сохранённое состояние: перемещает игрока и врагов, восстанавливает их здоровье и кулдаун игрока.

Архитектурно решение опирается на паттерн **Repository-Interactor**, что обеспечивает чёткое разделение ответственности и слабую связанность компонентов.

## Архитектурное решение: паттерн Repository-Interactor

В проекте введены следующие ключевые компоненты:

### 1. Модель данных `GameData`
Сериализуемые классы, описывающие состояние игрока и всех врагов. Хранят только данные, без логики.
```csharp
[System.Serializable]
public class GameData
{
    public PlayerData player;
    public List<EnemyData> enemies;
}
```

### 2. Репозиторий `IGameRepository` / `GameRepository`
Отвечает за физическое сохранение и загрузку данных. Использует существующий сервис `ISaveService` (работа с файлами через `PlayerPrefs` или JSON). Репозиторий инкапсулирует детали работы с хранилищем.
```csharp
public interface IGameRepository
{
    void Save(GameData data);
    GameData Load();
    bool HasSave();
}
```

### 3. Интерактор `GameInteractor`
Содержит бизнес-логику сохранения/загрузки. Получает из репозитория данные, преобразует текущее состояние игровых объектов (`GameModel`) в `GameData` и обратно. Взаимодействует с моделью сцены (`GameModel`), которая хранит ссылки на игровые объекты в рантайме.
```csharp
public class GameInteractor
{
    private readonly IGameRepository _repository;
    private readonly GameModel _model;

    public void SaveGame() { ... }
    public bool LoadGame() { ... }
}
```

### 4. Модель сцены `GameModel`
Простой контейнер с ссылками на активные игровые объекты (игрок, список врагов). Используется интерактором для получения данных и восстановления состояния.
```csharp
public class GameModel
{
    public GameObject Player { get; set; }
    public List<GameObject> Enemies { get; set; }
}
```

## Сохраняемые данные – детали

| Сущность | Сохраняемые параметры |
|----------|------------------------|
| Игрок | координаты `(x, y, z)`, текущее здоровье, оставшееся время кулдауна магии |
| Враг (любой тип) | тип врага (`Melee` или `EvilWatcher`), координаты `(x, y, z)`, текущее здоровье |
